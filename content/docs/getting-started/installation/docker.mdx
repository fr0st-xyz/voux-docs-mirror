## Docker Run

You can run Voux directly with Docker using a single command:

```bash title="Terminal"
docker run -d \
  --name voux \
  -p 8787:8787 \
  -e ADMIN_USERNAME=admin \
  -e ADMIN_PASSWORD=change-this-password \
  # -e PUBLIC_BASE_URL=https://your-domain.com \
  -v $(pwd)/data:/app/data \
  ghcr.io/quintixlabs/voux/voux:latest
```

<Callout title="Read this first" type="warning">
You must set `ADMIN_USERNAME` and `ADMIN_PASSWORD` to your own values. **Do not leave the example credentials!** 
<p />
If you run **Voux** on a public domain, set `PUBLIC_BASE_URL` to your full URL (for example, https://your-domain.com) so embeds and previews use the correct **HTTPS** address.
</Callout>

## Docker Compose

If you prefer **Docker Compose**, this is the simpler and recommended option for long running setups.

Create a `docker-compose.yml` file:

```yaml title="docker-compose.yml"
services:
  voux:
    image: ghcr.io/quintixlabs/voux/voux:latest
    container_name: voux # remove this if u wanna run more than 1 container
    ports:
      - "8787:8787"
    environment:
      - ADMIN_USERNAME=admin
      - ADMIN_PASSWORD=change-this-password
      # - PUBLIC_BASE_URL=https://your-domain.com
    volumes:
      - ./data:/app/data
```

<Callout title="Read this first" type="warning">
You must set `ADMIN_USERNAME` and `ADMIN_PASSWORD` to your own values. **Do not leave the example credentials!** 
<p />
If you run **Voux** on a public domain, set `PUBLIC_BASE_URL` to your full URL (for example, https://your-domain.com) so embeds and previews use the correct **HTTPS** address.
</Callout>


Then start it:

```bash title="Terminal"
docker compose up -d
```
